* 주제: 자율주행 데이터(LiDAR 포인트 클라우드와 카메라 이미지) 정렬
* 목적: 
  - 서로 다른 위치에 설치된 LiDAR 센서와 카메라 센서 간의 좌표계 변환 과정을 이해합니다.

### 3. 제공된 베이스라인 코드 및 파라미터 구조 분석 (2026-06-25 완료)

과제 해결을 위해 `problem_1.py`에 정의된 핵심 캘리브레이션 변수들과 데이터 전처리 과정을 분석했습니다.

#### A. 캘리브레이션 파라미터(Calibration Parameters) 분석
* **`R` & `t` (C0 → LiDAR Extrinsic)**
  - `R`: 카메라 0번(C0) 좌표계를 기준으로 LiDAR(L) 좌표계가 3차원 공간상에서 얼마나 회전되어 있는지 나타내는 3x3 행렬입니다.
  - `t`: 카메라 0번 원점으로부터 LiDAR 원점까지의 상대적인 거리를 나타내는 3x1 평행이동 벡터입니다.
* **`R_02` & `T_02` (C0 → C2 Extrinsic)**
  - `R_02`: 기준 카메라인 카메라 0번(C0)과 우측에 장착된 컬러 카메라인 카메라 2번(C2) 사이의 상대적인 회전 관계를 나타내는 3x3 행렬입니다.
  - `T_02`: 카메라 0번 원점으로부터 카메라 2번 원점까지의 물리적 거리를 나타내는 3x1 평행이동 벡터입니다.
* **`P` (C2 Intrinsic & Projection)**
  - 카메라 2번(C2)의 렌즈 초점거리와 주점, 그리고 보정(Rectification) 정보가 결합된 3x4 크기의 투영 행렬입니다.
   3차원 카메라 좌표계의 점을 우리가 보는 2D 이미지 평면(픽셀 좌표)으로 변환할 때 사용됩니다.

#### B. 데이터 로드 및 전처리 구조 분석
* **LiDAR 바이너리 데이터 로드 (`lidar_dtype` 및 `np.fromfile`)**
  - LiDAR가 저장하는 `.bin` 파일은 이진(Binary) 데이터 구조입니다. 데이터 하나당 `[x, y, z, intensity(반사도)]`가 각각 4바이트 실수형(`np.float32`)으로 연속해서 붙어 있습니다. 
  - 이를 파이썬에서 깨지지 않고 구조적으로 끊어 읽을 수 있도록 구조체 타입(`lidar_dtype`)을 선언하여 메모리에 로드했습니다.
* **동차 좌표계 변환을 통한 전처리 행렬 생성 (`ptcloud`)**
  - 외부 파라미터 행렬들과 차원을 맞추기 위해 선형대수학의 **동차 좌표계(Homogeneous Coordinates)** 개념을 적용했습니다.
  - `np.stack`으로 추출한 3D 좌표 `[X, Y, Z]` 뒤에 `np.insert`를 사용하여 의도적으로 `1`을 붙여 `[X, Y, Z, 1]` 형태의 4차원 벡터들로 확장했습니다.
  - 마지막에 전치 행렬(`.T`) 연산을 적용하여, 앞단에 4x4 변환 행렬을 바로 곱할 수 있도록 **4xN 차원의 행렬(`ptcloud`)** 구조로 세팅을 완료했습니다.

### 4. 과제 빈칸 해결 (2026-06-25 완료)

#### A. C02L 행렬 조립 (Camera 0 → LiDAR) - 2026-06-25 완료
* **1. 빈칸 코드**
  - `C02L = np.column_stack((R, t))`
* **2. 역할**
  - Camera 0 기준의 3D 공간 점을 LiDAR 좌표계로 변환하는 3x4 외부 파라미터(Extrinsic) 행렬입니다. 최종 목표인 [LiDAR -> Camera 2] 변환 행렬을 유도하기 위한 필수 기초 재료입니다.
* **3. 구현 방법**
  - **수학적 구조:** 3x3 회전 행렬 `R` 옆에 3x1 평행이동 벡터 `t`를 가로로 결합한 [R | t] 형태를 구축해야 합니다.
  - **NumPy 해결책:** `R`은 2차원(3x3)이지만 `t`는 1차원(3,)이라 일반 결합 함수(`np.hstack`)를 쓰면 차원 에러(`ValueError`)가 발생합니다. 따라서 1차원 벡터 `t`를 자동으로 2차원 열 벡터(3x1)로 취급해 `R` 옆에 에러 없이 붙여주는 `np.column_stack()`을 사용합니다.

#### B. C02C2 행렬 조립 (Camera 0 → Camera 2) - 2026-06-25 완료
* **1. 빈칸 코드**
  - `C02C2 = np.column_stack((R_02, T_02))`
* **2. 역할**
  - Camera 0 기준의 3D 공간 점을 Camera 2 좌표계로 변환하는 3x4 외부 파라미터(Extrinsic) 행렬입니다. 마찬가지로 최종 목표인 [LiDAR -> Camera 2] 변환 행렬을 유도하기 위한 두 번째 필수 기초 재료입니다.
* **3. 구현 방법**
  - **수학적 구조:** 3x3 회전 행렬 `R_02` 옆에 3x1 평행이동 벡터 `T_02`를 가로로 결합한 [R_{02} | T_{02}] 형태를 구축해야 합니다.
  - **NumPy 해결책:** `R_02`는 2차원(3x3) 행렬이지만 `T_02`는 방향성이 없는 1차원 배열(`shape: (3,)`) 구조입니다. 두 변수의 차원이 일치하지 않아 일반 가로 결합 함수로는 에러가 발생하므로, 1차원 벡터를 자동으로 2차원 열 벡터(3x1)로 세워 붙여주는 `np.column_stack()`을 사용하여 구현했습니다.

  #### C. C22L 행렬 조립 (Camera 2 → LiDAR) - 2026-06-25 완료
* **1. 빈칸 코드**
  - `C22L = C02L @ np.linalg.inv(C02C2)`
* **2. 역할**
  - Camera 2 좌표계 기준의 3D 공간 점을 LiDAR 좌표계로 변환하는 최종 4x4 동차 변환 행렬(Homogeneous Transformation Matrix)입니다.
  - 하단의 `## Problem` 구역에서 LiDAR 포인트 클라우드를 Camera 2 좌표계로 변환할 때 기준 축이 되는 행렬입니다.
* **3. 구현 방법**
  - **수학적 구조:** 두 센서 간의 다이렉트 변환 행렬이 없으므로 기준점인 Camera 0을 징검다리 삼는 행렬 체인(Chain) 연산을 설계해야 합니다. Camera 2에서 Camera 0으로 역행하는 변환을 먼저 구한 뒤, Camera 0에서 LiDAR로 가는 변환을 연속으로 곱해주는 기하학적 결합 관계를 유도합니다.
  - **NumPy 해결책:** NumPy에서 행렬의 역행렬을 계산해 주는 `np.linalg.inv()` 함수를 사용하여 `C02C2`를 역행렬로 뒤집은 후, 두 2차원 행렬 간의 행렬 곱셈을 수행하는 `@` 연산자를 활용하여 `C02L @ np.linalg.inv(C02C2)` 형태로 최종 구현했습니다.

  ### 5. Problem 구역 연산 구현

  #### A. LiDAR 포인트들을 Camera 2 좌표계로 변환 -2026-06-25 완료
* **1. 구현 코드**
  - `ptcloud_cam2 = np.linalg.inv(C22L) @ ptcloud`
* **2. 역할**
  - LiDAR 센서 중심축 기준으로 측정된 3D 포인트 클라우드 행렬을 Camera 2 센서 좌표계 기준으로 통째로 축 변환하는 단계입니다.
* **3. 구현 방법**
  - **수학적 구조:** 우리가 가진 `ptcloud`는 4xN 차원의 동차 좌표계 구조(각 열이 [X, Y, Z, 1]^T)로 정렬되어 있는 LiDAR 기준의 3D 점들입니다.
   위에서 구한 `C22L`은 Camera 2에서 LiDAR로 가는 4x4 변환 행렬(TC2->L)이므로, LiDAR에서 Camera 2 좌표계로 데이터를 가져오려면 `C22L`의 역행렬(TC2->L)^{-1} = TL->C2을 `ptcloud` 앞에 행렬 곱셈(`@`) 해주어야 합니다.
  - **NumPy 해결책:** NumPy의 `np.linalg.inv()` 함수로 4x4 외부 파라미터 행렬의 역행렬을 계산하고, 파이썬의 행렬 곱셈 연산자(`@`)를 활용하여 내적 연산을 한 줄로 깔끔하게 구현했습니다.

  #### B. 카메라 좌표계 기준 후방 포인트 제거 - 2026-06-25 완료
* **1. 구현 코드**
  - `valid_mask = ptcloud_cam2[2, :] > 0`
  - `ptcloud_cam2 = ptcloud_cam2[:, valid_mask]`
* **2. 역할**
  - 360도를 전부 전방위로 스캔하는 LiDAR 데이터 중, 전방만을 촬영하는 카메라 렌즈 뒤편(Z <= 0)에 위치하여 이미지에 투영될 수 없는 불필요한 3D 포인트들을 필터링하여 제거하는 단계입니다.
* **3. 구현 방법**
  - **수학적 구조:** 카메라 좌표계에서 표준 기하학적 축 정의상 Z축은 카메라가 바라보는 정면(전방) 방향을 의미합니다. 따라서 Z > 0 영역에 있는 점들만 실제 카메라 화면에 잡히는 유효한 포인트입니다. 4 x N 행렬 구조인 `ptcloud_cam2`에서 Z 좌표에 해당하는 3번째 행(`ptcloud_cam2[2, :]`)의 값이 0보다 큰 요소들만 판별해 냅니다.
  파이썬 행렬은 인덱스가 0부터 시작하므로 0번째 행이 X, 1번째 행이 Y, 2번째 행이 Z 좌표를 의미합니다. 즉, 데이터가 X, Y, Z 순서대로 행이 구성되어 있기 때문에 인덱스 2가 Z 좌표가 되며, 4xN 행렬 구조인 `ptcloud_cam2`에서 Z 좌표에 해당하는 3번째 행(`ptcloud_cam2[2, :]`)의 값이 0보다 큰 요소(전방 영역)들만 판별해 내는 구조를 가집니다.
  - **NumPy 해결책:** 연산은 총 두 줄로 진행됩니다. 첫 번째 줄은 데이터를 바로 지우지 않고 모든 점의 전방 존재 여부를 확인해 `True` 또는 `False`로 구성된 체크리스트(불리언 마스크)를 생성합니다. 이때 뒤에서 진행할 이미지 바깥쪽 필터링 마스크와 혼동되지 않도록 전방의 '유효한' 데이터라는 의미를 담아 변수명을 `valid_mask`로 명시했습니다. 두 번째 줄은 이 마스크를 행렬의 열 인덱스에 대입(`[:, valid_mask]`)하는 '불리언 인덱싱(Boolean Indexing)'을 수행하여, `True`(합격자)인 열만 쏙쏙 골라내 행렬을 새로 압축하고 후방 유령 점들을 실제로 완전히 삭제합니다.

  #### D. 이미지 경계 바깥 외곽점 제거 (Outlier Removal) - 2026-06-25 완료
* **1. 구현 코드**
  - `img_mask = (u >= 0) & (u < w) & (v >= 0) & (v < h)`
  - `u = u[img_mask]`
  - `v = v[img_mask]`
* **2. 역할**
  - 2D 이미지 평면으로 투영이 완료된 픽셀 좌표들 중, 실제 카메라의 해상도 범위(가로 w, 세로 h)를 벗어나 이미지 화면 바깥 영역에 찍히는 무의미한 외곽 점(Outlier)들을 필터링하여 최종 유효한 매핑 데이터만 남기는 단계입니다.
* **3. 구현 방법**
  - **수학적 구조:** 2D 이미지의 픽셀 좌표계는 좌측 상단 (0,0)을 기점으로 시작하여 우측 하단 (w, h)까지로 제한됩니다. 따라서 유효한 가로 픽셀 좌표 u는 0 이상 w 미만(0 <= u < w)이어야 하며, 세로 픽셀 좌표 v는 0 이상 h 미만(0 <= v < h)이어야 합니다. 이 네 가지 수학적 범위를 동시에 모두 충족(교집합, AND 연산)하는 유효 픽셀 영역을 판별하는 구조를 가집니다.
  - **NumPy 해결책:** NumPy의 비트 연산자 `&`를 활용하여 네 가지 불리언 조건을 원소별(Element-wise)로 동시 비교하는 `img_mask`를 생성합니다. 그 후, 불리언 인덱싱을 통해 유효 범위 내의 True 값에 해당하는 데이터들만 쏙 골라내어 기존 픽셀 배열 `u`와 `v`를 최종 압축하는 방식으로 구현했습니다.

- **오류 현상:** 로컬 컴퓨터 터미널에서 실행 파일(`uv.exe`, `python3.14.exe`)을 구동할 때 `os error 4551 (애플리케이션 제어 정책 차단)` 빨간색 에러가 뜨며 실행이 강제 봉쇄됨.
- **오류 원인:** 윈도우 자체 보안 시스템(AppLocker 등)이 인증되지 않은 최신 파이썬 빌드 파일의 시스템 내부 접근을 바이러스성 활동으로 오인하여 차단함.
- **해결 방안:** 로컬 컴퓨터 보안망의 간섭을 완전히 피하기 위해, 웹 브라우저에서 독립적으로 돌아가는 구글의 클라우드 가상 환경인 **구글 코랩(Google Colab)**으로 소스 코드와 데이터를 이동하여 실행 환경을 정상화함.

## 2. 소스 코드 오류 원인 및 해결 방안

 ### [오류 1] 시각화 데이터 개수 불일치 오류 (`ValueError`)
- **오류 원인:** 이미지 바깥 영역의 점들을 제거하는 4번 필터링 단계에서 2D 픽셀 좌표(`u`, `v`) 배열만 잘려 나가고, 점들의 색상을 결정하는 깊이 데이터(`z`)는 처음 개수 그대로 유지되어 데이터 크기가 서로 맞지 않아 에러 발생.
- **해결 방안:** 필터링 마스크 적용 구역에 `z = z[img_mask]` 코드를 추가하여 2D 좌표와 깊이 데이터의 개수를 똑같이 일치시킴.
- **수정 코드:**
  ```python
  ## 4. 이미지 바깥에 있는 Point (Outlier) 제거
  img_mask = (u >= 0) & (u < w) & (v >= 0) & (v < h)
  u = u[img_mask]
  v = v[img_mask]
  z = z[img_mask]  # u, v 개수와 동일하게 깊이 벡터 데이터 크기 동기화
  ```

 ### [오류 2] 넘파이 참조 복사(Shallow Copy)로 인한 깊이 데이터 오염
- **오류 원인:** 3번 정규화 단계에서 `ptcloud_cam2` 행렬을 직접 나누기 연산(`/=`)할 때, 파이썬 넘파이의 메모리 참조 특성상 주소를 공유하던 깊이 변수(`z`)까지 동시 오염됨. 이로 인해 기하학적 스케일 차원이 붕괴하여 모든 라이다 점이 우측으로 밀리는 수직 장벽 현상 발생.
- **해결 방안:** 연산 수행 전, `.copy()` 함수를 사용하여 완전히 독립된 메모리 공간에 깊이 데이터를 '깊은 복사(Deep Copy)'하여 데이터 원본을 안전하게 보호함.
- **수정 코드:**
  ```python
  ## 2. 카메라 좌표계에 맞게 뒤를 보는 부분 제거
  valid_mask = ptcloud_cam2[2, :] > 0
  ptcloud_cam2 = ptcloud_cam2[:, valid_mask]

  # [중요] 주소 참조 오염을 막기 위해 원본 Z(깊이) 데이터를 .copy()로 독립 메모리에 복사
  z = ptcloud_cam2[2, :].copy() 

  ## 3. Normalization (이제 원본을 깎아도 z는 안전함)
  ptcloud_cam2[0, :] /= ptcloud_cam2[2, :]
  ptcloud_cam2[1, :] /= ptcloud_cam2[2, :]
  ```

 ### [오류 3] 3x4 투영 행렬(P)의 행렬 곱 연산 누락
- **오류 원인:** 제공된 `P` 행렬은 단순 3x3 크기가 아니라 평행이동 상수까지 포함된 3x4 크기의 Projection 행렬임. 수식을 임의로 쪼개서 대입하는 과정에서 4번째 열 상수들이 연산에서 유실되어 좌표가 계속 왜곡됨.
- **해결 방안:** 수식을 강제로 분해하지 않고, 3x4 `P` 행렬과 `ptcloud_cam2` 행렬을 정석대로 먼저 일괄 행렬 곱(`@`) 처리한 뒤, 그 결과의 3번째 행 값으로 1행과 2행을 나누어 올바른 2D 픽셀 좌표를 도출함.
- **수정 코드:**
  ```python
  ## 3. Normalization (3x4 P 행렬과 호모지니어스 일괄 행렬곱 수행 후 Scale Factor 분할)
  ptcloud_img = P @ ptcloud_cam2
  u = ptcloud_img[0, :] / ptcloud_img[2, :]  
  v = ptcloud_img[1, :] / ptcloud_img[2, :]
  ```

 ### [오류 4] 외재 변수 결합 행렬 곱 순서 오류
- **오류 원인:** 모든 투영 수식을 맞추었음에도 똑같은 장벽 그림이 나왔던 결정적인 원인. 베이스 코드 내의 카메라-라이다 결합 식에 행렬 곱 순서 오류가 있었음. 행렬 곱은 앞뒤 순서가 바뀌면 완전히 다른 축으로 발산하기 때문에 라이다 축이 90도 옆구리로 뒤틀려 있었던 것임.
- **해결 방안:** 주석 제어 구역에 적혀 있던 역순 결합 식을 기하학적 정방향 순서인 `np.linalg.inv(C02C2) @ C02L`로 변경하여 뒤틀린 공간 축을 원점으로 똑바로 돌려놓음.
- **수정 코드:**
  ```python
  ## =================================================================
  ## C2 -> L Extrinsic Parameter (주석 제어 구역 수정)
  ## =================================================================
  C22L = np.linalg.inv(C02C2) @ C02L
  ## =================================================================
  ```
